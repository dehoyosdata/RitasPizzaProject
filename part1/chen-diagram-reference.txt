                   M..1/          1..1              \
                     /              |                \
                [EMPLOYEE]      [BRANCH]        [INSPECTION]
                /  |  |  \                       /  |  |  \
               /   |  |   \                     /   |  |   \
   (_employee_id_) | (wage) \        (_inspection_id_) |  (notes)
              (name) (hire_date)           (insp_date) (result)
                |
             (type)
               |
     {GMGR, Shift Manager,
      Cook/Kitchen, Cashier/Front}

    * Employee to Branch via WorksAt: TOTAL participation (double line)
      Every employee must be assigned to exactly one branch.
    * Employee to Branch via Manages: PARTIAL participation
      Only employees with type=GMGR participate. Each GMGR manages exactly one branch.

                  CUSTOMER — REWARD — ORDER (Owns + Redeems)
================================================================================

              [CUSTOMER]
                   |  1..N
                   |
                <Owns>
                   |
                 1..1
                   |
               [REWARD]
              /  |  |  \
             /   |  |   \
  (_reward_id_)  | (issue_date) (used_status)
        (description) (reward_type)
                   |
                 0..1
                   |
              <Redeems>
                   |
                 0..1
                   |
            [PIZZA_ORDER]

    * Owns: A customer owns many rewards; each reward belongs to one customer.
      Total participation on Reward side (every reward has an owner).
    * Redeems: A reward may be redeemed on at most one order (0..1).
      An order may redeem at most one reward (0..1).
    * Only registered customers can redeem rewards (BR13).
